Array

Jump to bottom
Ruslan edited this page on Mar 21, 2015 · 6 revisions

Array is a data type composed of a collection of (key,value) pairs, such that each possible key appears at most once in the collection. In comparison to the array implementations in other programming languages, array data type in Histone is almost the same as PHP's associative array. Arrays are ordered collection of items (that makes it different from JavaScript arrays, where the order is not guaranteed). An array can be created using the [] language construct:

{{* array with implicit key specification *}}
{{var cities = ["Moscow", "New York", "Paris", "London"]}}

{{* array with explicit key specification *}}
{{var capitals = ["RU": "Moscow", "US": "Washington", "NL": "The Hague"]}}

{{* you can mix implicit and explicit key specifications *}}
{{var concepts = ["fruits": "apples, oranges", "milk", "motorcycle": "hoe"]}}

{{* numbers, strings and identifiers are the only valid keys *}}
{{var novels = ["postman", 42: "deep thought", "1984": "ingsoc", swan: "song"]}}

{{* you can initialize array with any valid expression's values *}}
{{[
    key1: undefined,
    key2: null,
    key3: true,
    key4: false,
    key5: 123,
    key6: 2 + 2,
    key7: "string",
    key8: [1, 2, 3],
    key9: [
        foo: ["1", "2", "3"],
        bar: [x: 'y', 'z': 99]
    ]
]}}

Array elements can be accessed in a couple of different ways: using the array[key] syntax (for numeric and string keys), or using the array.key syntax (only when the key can be expressed as a valid identifier):

{{* array element with the key: 1 = "New York" *}}
{{["Moscow", "New York", "Paris", "London"][1]}}

{{* array element with the key: 1 = "1111" *}}
{{[1: '1111', "foo": '2222', x: "3333", "mmm": "4444"][1]}}

{{* array element with the key: foo = "2222" *}}
{{[1: '1111', "foo": '2222', x: "3333", "mmm": "4444"]["foo"]}}

{{* array element with the key: x = "3333" *}}
{{[1: '1111', "foo": '2222', x: "3333", "mmm": "4444"].x}}

{{* array element with the key: mmm = "4444" *}}
{{[1: '1111', "foo": '2222', x: "3333", "mmm": "4444"]["mmm"]}}

Any valid JSON identifier can be used as an array key:

{{var array = [if: "if", for: "for", return: "return"]}}
<div>{{array.if}} = if</div>
<div>{{array.for}} = for</div>
<div>{{array.return}} = return</div>

Leading and trailing commas are omitted:

{{[,,,,1, 2, 3,,,,]->toJSON}} = [1,2,3]

Commas between array items are also omitted if there is no valid expression (or key:value pair) between them:

{{[1,,,,,2,,,,,3]->toJSON}} = [1,2,3]

Please note that accessing array item by it's key and calling a method defined for the array, is completely different things, so different operators must be used in order to achieve desired result:

{{* returns array item with key "size" *}}
<div>{{[size: 30].size}} = 30</div>
{{* returns array item with key "size" *}}
<div>{{[size: 30]['si' + 'ze']}} = 30</div>
{{* calls array's method "size" *}}
<div>{{[size: 30]->size}} = 1</div>

Every value of the array data type is an object, which means that:

  • it can be used as operands for some operators (array concatenation using "+" operator for example)
  • it can have special predefined methods (join and slice for example), that you can call using "->" operator
{{* concatenate two arrays using "+" operator *}}
<div>{{([1, 2, 3] + [4, 5, 6])->toJSON}} = [1,2,3,4,5,6]</div>

{{* concatenate two arrays using "+" operator *}}
<div>{{([foo: "bar"] + [x: "y"])->toJSON}} = {"foo":"bar","x":"y"}</div>

{{* obtaining array size, using appropriate method *}}
<div>{{[1, 2, 3]->size}} = 3</div>

{{* obtain array of keys *}}
<div>{{[foo: "bar", x: "y"]->keys->toJSON}} = ["foo","x"]</div>

{{* obtain array of values *}}
<div>{{[foo: "bar", x: "y"]->values->toJSON}} = ["bar","y"]</div>

{{* join array elements into the string *}}
<div>{{[1, 2, 3]->join('-')}} = 1-2-3</div>

To check the value against array data type, you can use special method isArray that returns true in case if the value is an array and false in any other case:

{{* 10 - is not a array *}}
<div>{{10->isArray}} = false</div>

{{* "string" - is not a array *}}
<div>{{"string"->isArray}} = false</div>

{{* [1, 2, 3] - is a array *}}
<div>{{[1, 2, 3]->isArray}} = true</div>

{{* [foo: "bar", x: "y"] - is a array *}}
<div>{{[foo: "bar", x: "y"]->isArray}} = true</div>

When converting to string, array value is represented as a space-separated sequence of it's values:

{{* implicit conversion *}}
<div>{{[1, 2, 3]}}</div>
{{* explicit conversion *}}
<div>{{[bar: 'foo', foo: 'bar']->toString}}</div>

When converting to boolean, array value is represented as true (no matter how many elements it contains, so even empty array will be converted to true):

{{* implicit conversion *}}
<div>{{[1, 2, 3] ? 'true' : 'false'}} = true</div>
<div>{{[] ? 'true' : 'false'}} = true</div>
{{* explicit conversion *}}
<div>{{[foo: 'bar']->toBoolean}} = true</div>
<div>{{[]->toBoolean}} = true</div>

When converting to JSON, array value is represented as JSON-array or JSON-object, depending on it's contents, empty array is always converted into emtpy JSON-array:

<div>{{[1, 2, 3]->toJSON}} = [1, 2, 3]</div>
<div>{{[foo: 1, bar: 2]->toJSON}} = {"foo":1,"bar":2}</div>
<div>{{[]->toJSON}} = []</div>

[] operator Returns true if the value is integer, returns false in any other case
size Returns true if the value is integer, returns false in any other case
toLowerCase Returns true if the value is float, returns false in any other case
toUpperCase Returns the absolute value of a number
split Returns the absolute value of a number
charCodeAt Returns the absolute value of a number
toNumber Returns the absolute value of a number
strip Returns the absolute value of a number
